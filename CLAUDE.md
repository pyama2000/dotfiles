# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

Personal dotfiles for macOS and Linux, driven Nix-first (**no cargo-make**). macOS: nix-darwin + home-manager (flake, aarch64-darwin). Linux: standalone home-manager. Two entry-point scripts: `script/install.sh` bootstraps a new machine, `script/update.sh` performs the day-to-day update. CLI tools have a single source of truth — **Nix** (home-manager `home.packages` / `programs.*`) — whenever the tool exists in nixpkgs; `aqua` and `Homebrew` only cover what nixpkgs can't (see Conventions for the full layering order).

## Commands

Setup (new machine, idempotent — safe to re-run):

```sh
GIT_NAME=<name> GIT_EMAIL=<email> ./script/install.sh
# or omit the env vars and answer the interactive prompts
```

`install.sh` installs Homebrew + Nix, does the first `darwin-rebuild switch` (macOS) or `home-manager switch` (Linux), writes `~/.config/git/local/config` (read by `nix/home-manager/git.nix`), then handles what Nix can't: `aqua i`, `rustup`, `asdf` (nodejs/python), and on macOS lima VMs (`setup_lima`).

Day-to-day update:

```sh
./script/update.sh
```

Refuses to run with uncommitted changes. Order: `git pull --rebase origin main` → `nix flake update` → build check (`darwin-rebuild build` / `nix build .#homeConfigurations...`) → **auto-commits and pushes `nix/flake.lock` to main if changed** → `darwin-rebuild switch` / `home-manager switch` → `aqua i` / `rustup update` / `asdf plugin update --all` → macOS: `brew update && brew upgrade` → Neovim plugin update (`Lazy! update`, `MasonUpdate`) → **auto-commits and pushes `nvim/lazy-lock.json` to main if changed** → macOS: re-dumps `Brewfile` via `brew bundle dump --force --all` for manual review/commit (not auto-committed).

Apply Nix config manually (macOS):

```sh
cd nix
darwin-rebuild build --flake .#macos   # evaluate/build only (no changes applied)
darwin-rebuild switch --flake .#macos  # build and apply
```

Apply Nix config manually (Linux, standalone home-manager):

```sh
home-manager switch --flake ./nix#takahiko-yamashita@<x86_64|aarch64>-linux
```

Note: Nix flakes only include git-tracked files, so a newly added `nix/**.nix` file must be `git add`-ed (at least intent-to-add) before `darwin-rebuild`/`home-manager` can see it.

## Lint & format

CI (`.github/workflows/check.yaml`) gates PRs on these — run them locally before pushing:

- Lua: `stylua .` to format, `stylua --check .` to verify (config: `stylua.toml` — 2-space indent, 120 cols). Always run `stylua` after editing `.lua` files.
- GitHub workflows: `actionlint`
- Shell scripts: `shellcheck`

`/check` runs all of these at once.

Nix files are **not** gated by CI, but format them with `nixfmt` (run `nix fmt` from `nix/`, or `nixfmt <file>`) after editing `.nix` files.

## Conventions

- Commits: Conventional Commits — `type(scope): summary` (e.g. `feat(nvim): add telescope keymap`, `fix(tmux): correct prefix key`).
- Do **not** hand-edit `nvim/lazy-lock.json` — managed by `script/update.sh` (auto-committed) and Renovate.
- Do **not** hand-edit `nix/flake.lock` — managed by `script/update.sh` (auto-committed) or run `nix flake update` manually (in `nix/`).
- Tool source-of-truth order — check in this order before adding a CLI tool:
  1. **nixpkgs** via home-manager `home.packages` / `programs.*` (`nix/home-manager/default.nix`) — the default for any CLI tool.
  2. **aqua** (`aqua.yaml`) — only tools absent from nixpkgs (currently `tracer`, `lambroll`, `github/copilot-cli`).
  3. **Homebrew** via the nix-darwin `homebrew` module (`nix/darwin/homebrew.nix`) — `git` (bootstrap), `python@3.9` (EOL, dropped from nixpkgs), `readline`/`xz` (asdf python build deps), the `aqua` binary itself (not in nixpkgs), and GUI casks.
  4. **Native managers** for what's left: `rustup`; `asdf` for nodejs/python (no Nix auto-toolchain equivalent); a handful of cargo/uv/npm/go tools not in nixpkgs (e.g. `kani-verifier`, `awscli-local`, `cronv`, `sshtunnel`, MCP servers, `@playwright/cli`).
  When you move a tool into Nix, remove its `aqua.yaml`/`homebrew.nix` entry so it isn't double-managed.
- `Brewfile` is a reference mirror only, auto-regenerated by `script/update.sh` via `brew bundle dump --force --all` — **never hand-edit it**; `nix/darwin/homebrew.nix` is the real source of truth for Homebrew.
- Dotfile configs are symlinked by **home-manager** via `config.lib.file.mkOutOfStoreSymlink` (points at the repo, so edits apply without a rebuild), declared in `nix/home-manager/default.nix`: `nvim`, `wezterm`, `ghostty` (darwin only), `gh/config.yml` (file-level — `hosts.yml` stays local/untracked since it holds auth), `gh-dash`, `.tmux.conf`, `.profile`, `.vimrc`.
- `unfree`-licensed packages (`packer`: BSL 1.1, `ngrok`: proprietary) are allow-listed by name in the `unfreePackages` let-binding in `nix/flake.nix`, shared by macOS (via `specialArgs` into `nix/darwin/default.nix`) and Linux (`mkLinuxHome`).

## Layout

- `nvim/` — Neovim config (Lua, lazy.nvim). `lua/config/` for settings/keymaps, `lua/plugins/` for plugin specs.
- `script/` — just `install.sh` (bootstrap) and `update.sh` (daily update). No cargo-make, no `script/tasks/`.
- `nix/` — `flake.nix` (darwin + Linux home-manager outputs, `unfreePackages`) + `darwin/` (nix-darwin: `default.nix` for `system.defaults`/nix settings/unfree predicate, `homebrew.nix` for brews/casks, `fonts.nix`) + `home-manager/` (`default.nix` for `home.packages` + dotfile symlinks + docker cli-plugins, `git.nix`, `fish.nix`, `starship.nix`).
- `aqua.yaml` — CLI tools absent from nixpkgs only. `Brewfile` — auto-generated reference mirror only; the real Homebrew source of truth is `nix/darwin/homebrew.nix`. The bulk of CLI tools live in `nix/home-manager/default.nix` (`home.packages`).
- `ghostty/`, `gh/`, `gh-dash/`, `wezterm/`, `.tmux.conf`, `.profile`, `.vimrc` — symlinked terminal/tool config (see Conventions).

There is no `lima/` directory — lima VMs are created imperatively by `script/install.sh` (`setup_lima`, from `template://docker-rootful`); the file-descriptor limit they need is declared via `launchd.daemons."limit-maxfiles"` in `nix/darwin/default.nix`.
