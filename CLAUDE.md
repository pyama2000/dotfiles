# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

Personal dotfiles for macOS and Linux, driven Nix-first (**no cargo-make**). macOS: nix-darwin + home-manager (flake, aarch64-darwin). Linux: standalone home-manager. Two entry-point scripts: `script/install.sh` bootstraps a new machine, `script/update.sh` performs the day-to-day update. CLI tools have a single source of truth ‚Äî **Nix** (home-manager `home.packages` / `programs.*`) ‚Äî whenever the tool exists in nixpkgs; `aqua` and `Homebrew` only cover what nixpkgs can't (see Conventions for the full layering order).

## Commands

Setup (new machine, idempotent ‚Äî safe to re-run):

```sh
GIT_NAME=<name> GIT_EMAIL=<email> ./script/install.sh
# or omit the env vars and answer the interactive prompts
```

`install.sh` installs Homebrew + Nix, does the first `darwin-rebuild switch` (macOS) or `home-manager switch` (Linux), writes `~/.config/git/local/config` (read by `nix/home-manager/git.nix`), then handles what Nix can't: `aqua i`, `rustup`, `mise install` (nodejs/python, declared in `programs.mise.globalConfig`), and on macOS lima VMs (`setup_lima`).

Day-to-day update:

```sh
./script/update.sh
```

Refuses to run with uncommitted changes. Order: `git pull --rebase origin main` ‚Üí `nix flake update` ‚Üí build check (`darwin-rebuild build` / `nix build .#homeConfigurations...`) ‚Üí **auto-commits and pushes `nix/flake.lock` to main if changed** ‚Üí `darwin-rebuild switch` / `home-manager switch` ‚Üí `aqua i` / `rustup update` / `mise upgrade` ‚Üí macOS: `brew update && brew upgrade` ‚Üí Neovim plugin update (`Lazy! update`) ‚Üí **auto-commits and pushes `nvim/lazy-lock.json` to main if changed** ‚Üí macOS: re-dumps `Brewfile` via `brew bundle dump --force --formulae --casks --taps --go --cargo --uv --npm` (Homebrew 6 removed `--all`) for manual review/commit (not auto-committed).

Apply Nix config manually (macOS):

```sh
cd nix
darwin-rebuild build --flake .#macos   # evaluate/build only (no changes applied)
darwin-rebuild switch --flake .#macos  # build and apply
```

Apply Nix config manually (Linux, standalone home-manager):

```sh
home-manager switch --flake ./nix#takahiko-yamashita@<x86_64|aarch64>-linux
```

Note: Nix flakes only include git-tracked files, so a newly added `nix/**.nix` file must be `git add`-ed (at least intent-to-add) before `darwin-rebuild`/`home-manager` can see it.

## Lint & format

CI (`.github/workflows/check.yaml`) gates PRs on these ‚Äî run them locally before pushing:

- Lua: `stylua .` to format, `stylua --check .` to verify (config: `stylua.toml` ‚Äî 2-space indent, 120 cols). Always run `stylua` after editing `.lua` files.
- GitHub workflows: `actionlint`
- Shell scripts: `shellcheck`

`/check` runs all of these at once.

Nix files are **not** gated by CI, but format them with `nixfmt` (run `nix fmt` from `nix/`, or `nixfmt <file>`) after editing `.nix` files.

## Conventions

- Commits: Conventional Commits ‚Äî `type(scope): summary` (e.g. `feat(nvim): add telescope keymap`, `fix(tmux): correct prefix key`).
- Do **not** hand-edit `nvim/lazy-lock.json` ‚Äî managed by `script/update.sh` (auto-committed) and Renovate.
- Do **not** hand-edit `nix/flake.lock` ‚Äî managed by `script/update.sh` (auto-committed) or run `nix flake update` manually (in `nix/`).
- Tool source-of-truth order ‚Äî check in this order before adding a CLI tool:
  1. **nixpkgs** via home-manager `home.packages` / `programs.*` (`nix/home-manager/default.nix`) ‚Äî the default for any CLI tool.
  2. **aqua** (`aqua.yaml`) ‚Äî only tools absent from nixpkgs (currently `tracer`, `lambroll`, `github/copilot-cli`).
  3. **Homebrew** via the nix-darwin `homebrew` module (`nix/darwin/homebrew.nix`) ‚Äî `git` (bootstrap), `python@3.9` (EOL, dropped from nixpkgs), `readline`/`xz` (python build deps for mise's compile fallback and python@3.9), the `aqua` binary itself (not in nixpkgs), and GUI casks.
  4. **Native managers** for what's left: `rustup`; a handful of cargo/uv/npm/go tools not in nixpkgs (e.g. `kani-verifier`, `awscli-local`, `cronv`, `sshtunnel`, MCP servers, `@playwright/cli`). nodejs/python versions are managed by `mise` (the binary itself comes from nixpkgs via `programs.mise`; runtimes are installed by `mise install`).
  When you move a tool into Nix, remove its `aqua.yaml`/`homebrew.nix` entry so it isn't double-managed.
- `Brewfile` is a reference mirror only, auto-regenerated by `script/update.sh` via `brew bundle dump --force` with explicit category flags (`--formulae --casks --taps --go --cargo --uv --npm`; Homebrew 6 removed `--all`) ‚Äî **never hand-edit it**; `nix/darwin/homebrew.nix` is the real source of truth for Homebrew.
- **All tool configs live in the repo and apply without a rebuild.** Two mechanisms:
  - Symlinked via `config.lib.file.mkOutOfStoreSymlink` (declared in `nix/home-manager/default.nix` unless noted): `nvim`, `wezterm`, `github-cli/gh/config.yml` (file-level ‚Äî `hosts.yml` stays local/untracked since it holds auth), `github-cli/gh-dash`, `mise/config.toml`, `atuin/config.toml`, `herdr/config.toml`, `starship/starship.toml` (declared in `starship.nix`), `.tmux.conf`, `.profile`, `.vimrc`.
  - Sourced/included from generated files: fish sources `fish/interactive.fish` (+ `fish/functions/` via `fish_function_path`); git includes `git/config` (which references `git/ignore`, `git/commit` and includes `~/.config/git/local/config` last). The `programs.*` modules stay enabled only for package install and fish-init injection ‚Äî keep their `settings`/`globalConfig` empty, or they would conflict with the symlinks.
- `unfree`-licensed packages (`packer`: BSL 1.1, `ngrok`: proprietary) are allow-listed by name in the `unfreePackages` let-binding in `nix/flake.nix`, shared by macOS (via `specialArgs` into `nix/darwin/default.nix`) and Linux (`mkLinuxHome`).

## Layout

- `nvim/` ‚Äî Neovim config (Lua, lazy.nvim). `lua/config/` for settings/keymaps, `lua/plugins/` for plugin specs.
- `script/` ‚Äî just `install.sh` (bootstrap) and `update.sh` (daily update). No cargo-make, no `script/tasks/`.
- `nix/` ‚Äî `flake.nix` (darwin + Linux home-manager outputs, `unfreePackages`) + `darwin/` (nix-darwin: `default.nix` for `system.defaults`/nix settings/unfree predicate, `homebrew.nix` for brews/casks, `fonts.nix`) + `home-manager/` (`default.nix` for `home.packages` + dotfile symlinks + docker cli-plugins, `git.nix`, `fish.nix`, `starship.nix`).
- `aqua.yaml` ‚Äî CLI tools absent from nixpkgs only. `Brewfile` ‚Äî auto-generated reference mirror only; the real Homebrew source of truth is `nix/darwin/homebrew.nix`. The bulk of CLI tools live in `nix/home-manager/default.nix` (`home.packages`).
- `github-cli/` (gh + gh-dash), `wezterm/`, `starship/`, `mise/`, `atuin/`, `herdr/` (config.toml ‚Äî terminal multiplexer, replaces tmux for daily use; wezterm launches `herdr` on startup, tmux is kept installed as fallback), `git/` (config, ignore, commit template), `fish/` (interactive.fish + functions/), `.tmux.conf`, `.profile`, `.vimrc` ‚Äî repo-resident tool config, applied without rebuild (see Conventions).

There is no `lima/` directory ‚Äî lima VMs are created imperatively by `script/install.sh` (`setup_lima`, from `template://docker-rootful`); the file-descriptor limit they need is declared via `launchd.daemons."limit-maxfiles"` in `nix/darwin/default.nix`.
